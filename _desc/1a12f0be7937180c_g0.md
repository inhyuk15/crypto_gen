The one-zero padding function ozp is used to adjust the length of an input string so that the total length becomes a positive multiple of _n_ bits. For _X ∈_ B<sup>_∗_</sup> , ozp( _X_ ) is defined as ozp( _X_ ) = _X_ if _|X|_ = _ℓn_ for some _ℓ ≥_ 1, and ozp( _X_ ) = _X ∥_ 10<sup>_n−_1</sup><sup>_−_(</sup><sup>_|X|_mod</sup><sup>_n_)</sup> otherwise. We note that ozp( _ε_ ) = 10<sup>_n−_1</sup> , and we also note that, in general, the function is not invertible. 

The bit-fixing functions fix0 and fix1 are used to fix the most significant bit of an input string to zero and one, respectively. For _X ∈_ B<sup>_∗_</sup> , fix0( _X_ ) is defined as fix0( _X_ ) = _X ∧_ 01<sup>_|X|−_1</sup> , and fix1( _X_ ) is defined as fix1( _X_ ) = _X ∨_ 10<sup>_|X|−_1</sup> , where _∧_ and _∨_ are the bit-wise AND operation, and the bit-wise OR operation, respectively.

### **3.2 Parameter Spaces** 

As the CAESAR submission we specify the parameter spaces of CLOC as follows. 

> **–** Blockcipher _E_ : AES-128 (AES with 128-bit key), or TWINE-80 (TWINE with 80-bit key). 

> **–** Nonce length _ℓN_ : For AES-128, _ℓN ∈{_ 64 bits (8 byte) _,_ 96 bits (12 bytes) _,_ 112 bits (14 bytes) _}_ , and for TWINE-80, _ℓN ∈{_ 32 bits (4 byte) _,_ 48 bits (6 bytes) _}_ .
